### ROI Calculation Models

**Net Present Value (NPV)**
```
NPV = Σ (Cash Flow_t / (1 + r)^t) - Initial Investment

Where:
- Cash Flow_t = Net benefit in year t
- r = Discount rate
- t = Time period
```

**Internal Rate of Return (IRR)**
```
Find r where: NPV = 0
```

**Payback Period**
```
Payback = Initial Investment / Annual Net Benefit
```

**Total Cost of Ownership (TCO)**
```
TCO = Initial Costs + Ongoing Costs (over N years) + Transition Costs

Initial: Hardware, software, implementation, training
Ongoing: Operations, maintenance, support, licensing
Transition: Migration, decommissioning, change management
```

## Operating Model Design

### IT Operating Model Options

**Centralized**
```
┌─────────────────────────────────────────┐
│           IT Organization               │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐      │
│  │ Dev │ │ Ops │ │ Sec │ │ Data│      │
│  └─────┘ └─────┘ └─────┘ └─────┘      │
└─────────────────────────────────────────┘

Pros: Consistency, economies of scale, governance
Cons: Slow, disconnected from business, bottleneck
```

**Decentralized (Federated)**
```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  Business A  │  │  Business B  │  │  Business C  │
│  ┌────────┐  │  │  ┌────────┐  │  │  ┌────────┐  │
│  │   IT   │  │  │  │   IT   │  │  │  │   IT   │  │
│  └────────┘  │  │  └────────┘  │  │  └────────┘  │
└──────────────┘  └──────────────┘  └──────────────┘

Pros: Business-aligned, fast, responsive
Cons: Duplication, inconsistency, higher cost
```

**Hybrid (Bimodal / Platform)**
```
┌─────────────────────────────────────────────────────┐
│              Central Platform Team                   │
│  [Platform] [Security] [Architecture] [Data]        │
└───────────────────────┬─────────────────────────────┘
                        │
        ┌───────────────┼───────────────┐
        │               │               │
┌───────▼───────┐ ┌─────▼─────┐ ┌───────▼───────┐
│  Product Team │ │Product Team│ │ Product Team │
│  (Embedded IT)│ │(Embedded IT)│ │ (Embedded IT)│
└───────────────┘ └───────────┘ └───────────────┘

Pros: Balance of consistency and speed
Cons: Coordination overhead, potential conflict
```

### Team Topologies

**Stream-Aligned Teams**
- Aligned to business value stream
- End-to-end ownership
- Cross-functional

**Platform Teams**
- Provide internal services
- Enable stream-aligned teams
- Reduce cognitive load

**Enabling Teams**
- Help teams adopt new capabilities
- Temporary engagement
- Cross-cutting expertise

**Complicated Subsystem Teams**
- Own complex technical components
- Deep specialization required
- Provide services to other teams

### Integration Patterns

**Absorption**
- Target fully integrated
- Systems and processes adopted
- Brand and identity absorbed

**Preservation**
- Target operates independently
- Minimal integration
- Separate systems maintained

**Symbiosis**
- Selective integration
- Best of both approaches
- Coordinated but distinct

**Transformation**
- Both organizations change
- New combined capabilities
- Significant redesign

### Communicating Technology Risk

**Risk Communication Framework**
```
Risk: [Clear description of the risk]
Impact: [Business impact if risk materializes]
Likelihood: [Probability assessment]
Current Status: [What we're doing now]
Mitigation Plan: [What we will do]
Resources Needed: [Ask, if any]
```

**Example**
```
Risk: Core banking system end-of-support in 18 months
Impact: Security vulnerabilities, compliance risk, $2M+ in custom fixes
Likelihood: Certain (vendor announced EOL)
Current Status: Modernization program initiated
Mitigation Plan: Replace with cloud-native platform by Q4 2025
Resources Needed: $5M investment, 15 additional engineers
```
